# Agent Result Contract

Every completed CoreKB agent task should report the sections below. If a task provides a stricter result format, use the task-specific format.

## Summary

Concise description of what changed.

## Files Changed

List created, modified, moved, or deleted files.

## Behavior Added

Explain behavior changes. For documentation-only tasks, state that no runtime behavior changed.

## Tests Run

List exact commands run.

## Test Result

Summarize pass/fail results and warnings that matter.

## Database Impact

State whether models, migrations, schemas, or data writes changed.

## Runtime Impact

State whether application runtime, ingestion, indexing, retrieval, frontend, or production config changed.

## Risk Notes

Call out uncertainty, skipped checks, environment limits, or review-sensitive areas.

## Rollback Notes

Explain how to revert the task safely.

## Open Questions

List unresolved decisions. Use `None` when there are no open questions.
